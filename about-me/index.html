关于博客

关于博客，就是一个技术类的博客，但是呢博客的深度跟博主现在的水平一样。不是很深，没关系嘛，慢慢来，我会一步一步深入，然后把这个博客的深度提升一个档次。 我喜欢的技术栈呢有 SpringBoot 和 SpringCloud 两大 Spring 全家桶。当然啦，ORM 层如果以前你问我喜欢用哪个，我可能想都没想就会跟你说 MyBatis。也因为如此给公司做的初步架构就是使用 MyBatis 来进行业务的开发的。但是 MyBatis 用得多了就会感觉其实很多时候我们的业务不需要要求那么高的响应速度的，使用 Hibernate 也许能够让代码看起来更加的面向对象，具有代码层次上的美。所以现在，在 ORM 层，如果这个业务要求不需要很高的并发量（除了下单之外其他基本不需要）我都会选用 Hibernate 来做项目的 ORM 层，这也让我更加爱上了 SpringDataJpa 这个项目。 我的目标呢，就是在我的博客里面，有 SpringBoot 和 SpringCloud 两大体系的系列教程，也许做教程的人有很多，而且他们写的都比我好，但是我还是想有一份属于我自己的系列博客，以前完成度比较高的还有 Thread 系列（JUC 包都还没做）以及 JDK8 系列。如果进来了这个博客，不容错过哦。 当然我现在也还想开始一些碎碎念的文章，可能是吐槽，也可能是一些想法，但是由于一些不可告人的原因，可能上了也可能没上，具体看我的文章分类而定。

关于博主

我的名字就叫 Weidan，一个 94 年出生的大专毕业小伙汁。这里我想还可以介绍我入海的经历： 好像是从高一开始的，家里因为买了电脑已经两年多了，加上我笨拙的手，导致我对 PC 游戏并不是很感兴趣（转而感兴趣的是任天堂系列的口袋游戏或者忍者游戏）。所以平时按照大人们的话说就是学习了一周了，开个电脑让你放松放松吧。然后我打开电脑也不知道干啥，有一次偶然看到有手机论坛刚好是我的型号的手机的（诺基亚5530）可以进行刷机，那像我这么爱折腾的肯定不放过啦，刷机，搞定。还不够满足，还需要修改固件的内容，所以我在高中的时候就经常爬帖，怎么修改固件啊等等。反正每次找到修改刷入手机可以正常使用就有一种成就感。但是还有更加好玩的玩法，是需要去修改一系列代码的，但是源码我们不可能看到，所以我们需要修改的一般是修改 16 进制码。那时候真的让我感觉，以后做程序猿需要面对的都是这些 16 进制码吗？然后就在我头脑里面埋下了拒绝成为程序员的想法。 但是奇怪的是，我所有科目就是数学最好了，可能脑袋还是喜欢逻辑性强的事物。 于是乎，折腾来折腾去的，折腾学习终于过去了，来到了高三报考志愿的时候。于是乎我跟我爸说了我坚决不做程序员，然后就报考了一个叫做物流管理但是毕业后去送快递送外卖的职业。现在想来想去，只能说，高中的时候只知道好好学习，学到毕业整个人都傻逼了，连自己以后想干嘛都不知道。 哈哈哈，生活就是喜欢跟人开玩笑。一切归根于我在大一的时候，想去专升本（就是所说的大自考）里面接触了 Java 这门课程。怎么学呢，自己看又看不到，问我学编程的同学，同学说你去搜索视频去看啊，于是乎找到了某智的视频，从此开始看，一发不可收拾。开始觉得原来代码也可以这么好玩，就越学越深入了 终于毕业的时候一家创业型公司收留了我，我出来就开始坑一次队友，就是架构做的不好，让大家工作得都很累。所以现在我感觉一个项目，先不聊什么前景啊什么东西，因为我是个技术员，那么我第一关心的还是这个项目的架构是否做的合适，开发起来是否舒服等等这些问题。这个业务场景我是否可以使用一个新的框架来让开发更加迅速，更加减少 BUG 的出现等等问题。 感谢后来的转型，使用了 SpringCloud 这个项目以及 VUE 前端结合开发，让项目开发起来更加顺畅，现在写的垃圾代码少了，业务逻辑代码多了，不妨也可以说是一种提高工作效率的方式。 这也许就是我工作两年来给我带来的收获吧，当然前方路还有一段，还是需要不停的进修。
